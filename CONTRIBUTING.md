# Contributing

Thank you for your interest in contributing! This project is open to contributions.

## How to Report Issues

Found a bug or have a feature request? Please open an issue:

1. Go to the [Issues](../../issues) tab
2. Click "New Issue"
3. Provide:
   - Clear title describing the problem/request
   - Detailed description of what happened
   - Steps to reproduce (for bugs)
   - Your environment (Windows version, Python version)
   - Error messages or logs if applicable

## Bug Reports

When reporting bugs, please include:

```
**Describe the bug:**
A clear description of what the bug is.

**To Reproduce:**
1. Step 1
2. Step 2
3. ...

**Expected behavior:**
What should happen

**Actual behavior:**
What actually happened

**Environment:**
- OS: Windows 11
- Python version: 3.11
- Backup size: ~50GB
- Date range: 2024-12-01 to 2024-12-31

**Logs/Error Messages:**
[Paste relevant output here]
```

## Feature Requests

When suggesting enhancements, explain:

```
**Use Case:**
Why would this feature be useful?

**Suggested Solution:**
Your idea for implementation

**Alternatives Considered:**
Other approaches you've thought about
```

## Development Setup

If you want to modify the code:

1. Clone the repository
2. Create a feature branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Test thoroughly
5. Submit a pull request with:
   - Clear description of changes
   - Reference to any related issues
   - Testing notes

## Code Style

- **Python**: Follow PEP 8
- **PowerShell**: Use consistent indentation and naming
- **Batch**: Keep comments clear and functions well-organized
- **Documentation**: Keep README and guides up-to-date

## Testing

Before submitting changes, please test with:

- Different backup sizes
- Various date ranges
- Both interactive and command-line modes (for applicable changes)
- Edge cases (empty backups, corrupted files, etc.)

## Documentation

If adding features, please update:
- README.md with usage examples
- Comments in code
- CHANGELOG.md (maintainer will do this at release)

## Pull Request Process

1. Ensure your code follows the style guidelines
2. Update documentation as needed
3. Test thoroughly
4. Create a pull request with a clear description
5. Wait for review and feedback

## Questions?

Feel free to open issues with questions. Even if they're not bugs or feature requests, they help improve the documentation and project.
